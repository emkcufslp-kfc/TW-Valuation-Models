# UI_CONTENT_PLAN_ZH_TW

## 文件目的
本文件規劃整合式台股估值平台的繁體中文 UI 內容架構，聚焦於：
- 頁面層級
- 區塊命名
- 中文文案風格
- 欄位顯示原則
- 狀態與錯誤訊息語氣

本文件不處理：
- 視覺稿像素細節
- CSS / 元件實作
- 路由技術選型

## 內容語氣原則
- 專業
- 清楚
- 節制
- 避免過度行銷語氣
- 以研究平台與決策輔助工具口吻呈現

## UI 語言原則
- 主要語言：繁體中文
- 英文保留給 ticker、來源名稱、技術欄位代碼、模型英文名
- 中文應優先使用台灣金融使用者熟悉的字詞

## 命名風格原則
- 頁面標題：偏正式、明確
- 區塊標題：短而有辨識度
- 按鈕文案：動作導向，不使用含糊動詞
- 錯誤文案：說明問題與可理解原因，不責怪使用者

## 建議資訊架構
1. 首頁總覽
2. 個股分析
3. 模型比較
4. 模型明細
5. 資料狀態
6. 系統說明

## 1. 首頁總覽
### 頁面標題候選
- 台股估值整合平台
- 台股估值研究儀表板
- 台股多模型估值總覽

建議主標題：
`台股估值研究儀表板`

### 副標題候選
- 整合多種估值模型，快速比較不同方法對同一檔股票的判讀
- 以共用資料層驅動，保留各模型原始邏輯與獨立觀點
- 在同一介面中追蹤估值、資料新鮮度與模型差異

建議副標題：
`整合多種估值模型，於同一介面比較不同方法對個股的判讀。`

### 首頁主要區塊
- 市場摘要
- 模型總覽
- 快速個股查詢
- 今日資料狀態
- 重要提示

### 市場摘要文案
區塊標題：
`市場摘要`

欄位名稱候選：
- 加權指數
- 大盤收盤
- 漲跌點數
- 漲跌幅
- 更新時間

### 模型總覽文案
區塊標題：
`模型總覽`

卡片層級建議欄位：
- 模型名稱
- 模型定位
- 最新資料日
- 可分析股票數
- 執行狀態
- 備註

模型定位範例：
- 價值投資取向
- 財務品質取向
- 混合估值取向
- 官方資料優先取向

### 快速個股查詢

`快速個股查詢`

### 今日資料狀態

`資料更新狀態`

欄位名稱：
- 價格資料

- 資料來源
- 最後更新

## 2. 個股分析頁
### 頁面標題格式
`{ticker} {company_name} 個股分析`

範例：
`2330 台積電 個股分析`

### 主要區塊
- 核心摘要
- 模型比較
- 關鍵指標
- 歷史價格與區間定位
- 風險與提醒

### 核心摘要
區塊標題：
`核心摘要`

建議欄位：
- 股票代號
- 公司名稱
- 收盤價
- 最新交易日
- 綜合觀察

綜合觀察的顯示原則：
- 不做單一總分
- 可用「多數模型偏低估 / 分歧大 / 資料不足」等中性語句

### 模型比較
區塊標題：
`模型比較`

表格欄位建議：
- 模型
- 估值判斷
- 合理價區間
- 相對現價
- 資料日期
- 來源摘要
- 備註

估值判斷文案候選：
- 偏低估
- 接近合理
- 偏高估
- 訊號分歧
- 資料不足
- 該模型未提供

### 關鍵指標
區塊標題：
`關鍵指標`

欄位候選：
- 本益比
- 股價淨值比
- 每股盈餘
- 每股淨值
- 營收成長率
- 毛利率
- 營業利益率
- ROE
- 自由現金流

### 歷史價格與區間定位
區塊標題：
`歷史價格與區間定位`

輔助文案：
`用於觀察目前價格在近期歷史區間中的位置，協助理解模型判讀背景。`

### 風險與提醒
區塊標題：
`風險與提醒`

文案範例：
- `部分資料欄位由替代來源補齊，請留意口徑差異。`
- `此檔個股部分季度資料不足，相關模型結果可信度較低。`
- `不同模型假設不同，結果不宜直接視為單一結論。`

## 3. 模型比較頁
### 頁面定位
用於橫向比較多模型對同一檔股票的輸出差異。

### 頁面標題
`多模型比較`

### 區塊設計
- 比較摘要
- 模型結果表
- 模型差異說明

### 比較摘要文案
區塊標題：
`比較摘要`

文案樣式建議：
- `多數模型落在合理至偏低估區間。`
- `模型之間差異明顯，建議進一步檢查資料日期與估值假設。`
- `目前可用模型不足，暫不建議進行橫向比較。`

### 模型差異說明
區塊標題：
`模型差異說明`

固定說明文案範例：
`各模型使用的估值邏輯、資料期間與指標權重不同，結果僅適合並列參考，不代表彼此可以直接互相替代。`

## 6. 系統說明頁
### 頁面標題
`系統說明`

### 區塊建議
- 平台定位
- 模型原則
- 資料來源原則
- 使用限制

### 平台定位文案
`本平台整合多個台股估值模型，目的是協助研究與比較，不是將不同模型壓縮成單一分數。`

### 模型原則文案
`各模型保留原始邏輯與獨立輸出，平台僅負責資料供應、結果呈現與比較。`

### 資料來源原則文案
`平台優先使用官方或較高可信度來源，必要時才會使用備援來源，並保留來源標記供使用者查閱。`

- `資料不足`

## 共用空狀態文案
- `目前尚無可顯示資料。`
- `請先輸入股票代號並執行分析。`
- `此模型暫時沒有可用輸出。`
- `目前資料不足，無法完成比較。`

## 共用錯誤訊息文案
- `目前無法完成此操作，請稍後再試。`
- `查無此股票代號，請確認輸入內容。`
- `資料來源暫時不可用，系統已嘗試備援來源。`
- `部分資料欄位缺漏，結果僅供參考。`

## 共用說明 tooltip 文案方向
- 以一句話說清楚欄位含義
- 優先解釋資料口徑與限制
- 避免過度教學式長段落

範例：
- `合理價區間：依該模型假設推算之價格區間，非保證價格。`
- `資料日期：此模型本次運算所使用的最新資料日期。`
- `使用備援：主要來源不可用時，改由替代來源補齊。`

## 視覺語氣對應提醒
為了配合已確認的深色專業風格，UI 文案應避免：
- 過度口語
- 過度熱血或行銷化
- 帶有明確買賣暗示的字眼

建議多使用：
- 觀察
- 判讀
- 區間
- 資料狀態
- 模型差異
- 研究參考
